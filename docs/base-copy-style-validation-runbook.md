# Base copy style validation runbook

This runbook validates that Test Mail Orchestrator copy variety starts at the base copy origin, before fingerprint reserve/mirror variants are applied.

## Why this exists

PowerEmail test messages were still visually clustering around a soft outreach pattern such as:

- `Un detalle sobre ...`
- `Una consulta sobre ...`
- short contextual observation
- implicit business/prospecting tone

PR #33 introduced deterministic base copy styles in `src/test-mail-orchestrator.js`. This runbook is the operational closeout path for confirming the deployed behavior and detecting regressions later.

## Scope

Applies to test automation traffic generated by `src/test-mail-orchestrator.js`.

This does not change production campaign copy behavior.

## Expected healthy behavior

A healthy preview or handoff should show:

- `baseCopyStyle` / `base_copy_style` populated
- `baseCopyStyleName` / `base_copy_style_name` populated
- `baseCopyMatrix` / `base_copy_matrix` populated
- mixed styles across slots or tenants
- some messages without questions
- some messages without CTA
- some messages without greeting
- some messages without traditional close
- visible differences in body length and paragraph count

Acceptable style examples include:

- `one_line_note`
- `phone_fragment`
- `operational_comment`
- `quiet_observation`
- `single_question`
- `long_personal_note`
- `formal_record`
- `acknowledgement`
- `internal_note`
- `neutral_clarification`

## Regression indicators

Treat the behavior as suspicious if recent base snapshots show:

- many subjects starting with the same pattern, for example `Un detalle sobre ...` or `Una consulta sobre ...`
- most bodies using the same length bucket
- most bodies using the same paragraph count
- repeated sales/outreach phrases such as `Hola`, `Vi que`, `Note que`, `Han considerado`, `Te interesaria`, `Si tiene sentido`, `Quedo atento`, `Saludos cordiales`
- missing `base_copy_style` metadata after PR #33 is deployed
- only fingerprint mirror variants show style metadata, while parent/base snapshots do not

## 2. Preview copy without sending

Use local copy first because it validates the deterministic base style system without OpenAI variability.

```bash
cd ~/poweremail-direct-dispatcher

railway run \
  --service poweremail-direct-dispatcher \
  --environment production \
  -- \
  env \
  TEST_ORCHESTRATOR_MODE=plan \
  TEST_ORCHESTRATOR_COPY_MODE=local \
  TEST_ORCHESTRATOR_PREVIEW_COPY=true \
  TEST_ORCHESTRATOR_PREVIEW_LIMIT=8 \
  TEST_ORCHESTRATOR_TENANT=lester \
  npm run orchestrate:test
```

Success criteria in `test_orchestrator.copy_preview`:

- preview rows include `baseCopyStyle`
- preview rows include `baseCopyStyleName`
- preview rows include `baseCopyMatrix`
- subjects and bodies are not all shaped as outreach-style consultation emails
- at least one preview is short or very short when multiple previews are available

If only one slot is planned for the selected tenant/day, run multiple tenants or omit `TEST_ORCHESTRATOR_TENANT` to inspect broader distribution.

Success criteria:

- `test_orchestrator.handoff_completed` appears
- parent/base dispatch is created or completed normally
- mirror dispatch either creates a valid variant or is intentionally blocked by fingerprint policy with persisted failure metadata
- the base snapshot itself has `base_copy_style` metadata

## 4. SQL verification for recent snapshots

```bash
psql "$PSQL_LEADS_URL" <<'SQL'
SELECT
  content_snapshot_id,
  dispatch_campaign_id,
  sendy_campaign_id,
  subject,
  plain_text,
  source_json->>'copy_source' AS copy_source,
  source_json->>'copy_topic' AS copy_topic,
  source_json->>'base_copy_style' AS base_copy_style,
  source_json->>'base_copy_style_name' AS base_copy_style_name,
  source_json->'base_copy_matrix' AS base_copy_matrix,
  source_json->>'fingerprint_variant' AS fingerprint_variant,
  source_json->>'fingerprint_variant_strategy' AS fingerprint_variant_strategy,
  created_at
FROM control_plane.campaign_content_snapshots
WHERE created_at > now() - interval '60 minutes'
ORDER BY content_snapshot_id DESC
LIMIT 20;
SQL
```

Success criteria:

- base/parent test snapshots have `base_copy_style` populated
- metadata is not limited to reserve mirror fingerprint variants
- `plain_text` shows style differences, not only topic substitutions

## 5. Distribution query

```bash
psql "$PSQL_LEADS_URL" <<'SQL'
SELECT
  source_json->>'base_copy_style' AS base_copy_style,
  source_json->>'base_copy_style_name' AS base_copy_style_name,
  count(*) AS snapshots,
  min(created_at) AS first_seen,
  max(created_at) AS last_seen
FROM control_plane.campaign_content_snapshots
WHERE created_at > now() - interval '24 hours'
  AND source_json->>'source_system' = 'poweremail-test-automation'
GROUP BY 1, 2
ORDER BY snapshots DESC, base_copy_style;
SQL
```

Success criteria:

- more than one `base_copy_style` appears over a multi-slot or multi-tenant window
- if only one style appears, confirm whether the window only contains one generated slot

## 6. Lightweight anti-outreach regression query

```bash
psql "$PSQL_LEADS_URL" <<'SQL'
SELECT
  content_snapshot_id,
  dispatch_campaign_id,
  subject,
  source_json->>'base_copy_style' AS base_copy_style,
  plain_text,
  created_at
FROM control_plane.campaign_content_snapshots
WHERE created_at > now() - interval '24 hours'
  AND source_json->>'source_system' = 'poweremail-test-automation'
  AND (
    plain_text ILIKE '%Quedo atento%'
    OR plain_text ILIKE '%Saludos cordiales%'
    OR plain_text ILIKE '%Te interesaria%'
    OR plain_text ILIKE '%Han considerado%'
    OR plain_text ILIKE '%Si tiene sentido%'
    OR plain_text ILIKE '%Vi que%'
    OR plain_text ILIKE '%Note que%'
    OR subject ILIKE 'Una consulta sobre%'
    OR subject ILIKE 'Un detalle sobre%'
  )
ORDER BY created_at DESC
LIMIT 20;
SQL
```

Success criteria:

- zero rows is ideal
- any rows should be reviewed as possible base copy strategy regression

## Operational close criteria

Consider the base copy style change healthy only when all are true:

- deployed service is online
- preview shows base style metadata
- handoff smoke creates a base snapshot with `base_copy_style`
- recent SQL confirms the metadata is present in `campaign_content_snapshots.source_json`
- copy shape varies by style rather than only by topic/synonym

## Known limitation

This validates the test orchestrator origin copy path. It does not prove Microsoft Defender recovery. Defender impact still requires 48-72 hours of placement evidence grouped by tenant, sender domain, recipient cohort, content style, and fingerprint variant behavior.
